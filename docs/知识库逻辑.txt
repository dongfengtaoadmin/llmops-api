分割利弊
Top‑K 就是“只取排名最靠前的 K 个结果”
二、两种分割方式对比
    情况 A：分割成 1000 个片段
    每个片段 = 1 句话

    总片段数 1000

    检索 10 个片段 → 10 句话

    情况 B：分割成 500 个片段
    每个片段 = 2 句话

    总片段数 500

    检索 10 个片段 → 20 句话

    不是“分割越少，检索到更多片段”
    而是：相同检索数量下，更粗的分割带来更多总信息量

  向量数据库存储的内容                                                                                                                                  
                                                                                                                                                        
  ┌──────────┬───────────────────────────────────────────┐                                                                                              
  │  存储项  │                 对应关系                  │                                                                                              
  ├──────────┼───────────────────────────────────────────┤                                                                                              
  │ 文本内容 │ 片段的 page_content（分割好的文本）       │                                                                                            
  ├──────────┼───────────────────────────────────────────┤                                                                                              
  │ 向量嵌入 │ 文本经过 Embedding 模型转换的向量         │                                                                                              
  ├──────────┼───────────────────────────────────────────┤                                                                                              
  │ ID       │ 片段的 node_id（唯一标识）                │                                                                                              
  ├──────────┼───────────────────────────────────────────┤                                                                                              
  │ 元数据   │ 片段的 metadata（segment_id、enabled 等） │                                                                                            
  └──────────┴───────────────────────────────────────────┘                                                                                              
                                                                                                                                                      
  数据流转                                                                                                                                              
                                                                                                                                                      
  原始文档                                                                                                                                              
      ↓ 解析                                                                                                                                            
  文档内容                                                                                                                                              
      ↓ 分割                                                                                                                                            
  片段列表                                                                                                                                              
      ↓ 存储                                                                                                                                            
  向量数据库（每个片段一条记录）                                                                                                                        
                                                                                                                                                        
  对应关系                                                                                                                                              
                                                                                                                                                        
  PostgreSQL                          向量数据库                                                                                                        
  ───────────────────────────────    ───────────────────────────────                                                                                    
  Segment 表:                                                                                                                                           
    id: segment_id (片段ID)                                                                                                                             
    node_id: "uuid-xxx"         ←──→  id: "uuid-xxx" (向量记录ID)                                                                                       
    content: "片段文本..."             text: "片段文本..."                                                                                              
                                       vector: [0.1, 0.2, ...]                                                                                          
                                                                                                                                                        
  为什么要这样设计？                                                                                                                                    
                                                                                                                                                        
  1. 检索时：用户提问 → 向量相似度匹配 → 找到最相关的片段                                                                                               
  2. 关联数据：通过 node_id 可以从向量数据库找到 PostgreSQL 中的片段详情（关键词、命中次数等）                                                        
                                                                                                                                                        
  简单总结                                                                                                                                              
                                                                                                                                                        
  一个片段 = 向量数据库中的一条记录，用 node_id 作为唯一标识来关联两个存储系统。  




  向量数据库记录                                                                                                 
  ──────────────────────────────                                                                                 
  id: "uuid-xxx"               # node_id                                                                         
  vector: [0.12, -0.34, ...]   # 文本向量（Embedding生成）                                                       
  text: "片段文本内容..."       # 原始文本                                                                       
  metadata: {                  # 元数据（一起存储）                                                              
      segment_id: "uuid-yyy",                                                                                    
      document_enabled: True,  # ← 你问的这些字段                                                                
      segment_enabled: True,   # ← 存在 metadata 里                                                              
      ...                                                                                                        
  }                                                                                                              
                                                                                                                 
  存储流程                                                                                                     
                                                                                                               
  # add_documents 会把整个 LCDocument 存进去
  LCDocument(                                                                                                    
      page_content="片段文本",    # → 存为 text + 生成 vector                                                    
      metadata={                 # → 整个 metadata 一起存储                                                      
          "segment_id": "xxx",                                                                                   
          "document_enabled": True,                                                                              
          "segment_enabled": True,                                                                               
      }                                                                                                          
  )                                                                                                              
                                                                                                                 
  # 调用 add_documents 后，全部存入向量数据库                                                                    
  vector_store.add_documents(chunks, ids=ids)                                                                    
                                                                                                                 
  检索时使用 metadata                                                                                            
                                                                                                                 
  # 可以用 metadata 进行过滤检索                                                                                 
  results = vector_store.similarity_search(                                                                      
      query="什么是机器学习？",                                                                                  
      k=5,                                                                                                       
      filter={                                                                                                   
          "document_enabled": True,    # 用 metadata 过滤                                                        
          "segment_enabled": True,                                                                               
      }                                                                                                          
  )                                                                                                              
       





 向量检索和关键词检索的场景
 从 retrieval_service.py 可以看到三种检索策略：                                                                      
   
  # 三种检索策略                                                                                                      
  if retrieval_strategy == RetrievalStrategy.SEMANTIC:                                                                
      lc_documents = semantic_retriever.invoke(query)      # 语义检索（向量数据库）                                   
  elif retrieval_strategy == RetrievalStrategy.FULL_TEXT:                                                             
      lc_documents = full_text_retriever.invoke(query)      # 全文检索（关键词表）                                    
  else:                                                                                                               
      lc_documents = hybrid_retriever.invoke(query)         # 混合检索（两者结合）                                    
                                                                                                                      
  向量数据库 vs 关键词表                                                                                              
                                                                                                                      
  ┌──────────┬───────────────────────────┬──────────────────────┐                                                     
  │  对比项  │        向量数据库         │       关键词表       │                                                   
  ├──────────┼───────────────────────────┼──────────────────────┤                                                     
  │ 检索方式 │ 语义相似度匹配            │ 关键词精确匹配       │ 
  ├──────────┼───────────────────────────┼──────────────────────┤                                                     
  │ 优势     │ 理解语义，找相关内容      │ 精确匹配，找特定词汇 │                                                     
  ├──────────┼───────────────────────────┼──────────────────────┤                                                     
  │ 劣势     │ 可能漏掉精确词汇          │ 无法理解同义词/语义  │                                                     
  ├──────────┼───────────────────────────┼──────────────────────┤                                                     
  │ 存储位置 │ 向量数据库（如 Weaviate） │ PostgreSQL（JSONB）  │                                                   
  └──────────┴───────────────────────────┴──────────────────────┘                                                     
                                                                  
  三种检索方式                                                                                                        
                                                                  
  用户问题："什么是人工智能？"                                                                                        
             ↓                                                                                                        
  ┌─────────────────────────────────────────────────┐                                                                 
  │              三种检索策略                         │                                                               
  ├─────────────────┬───────────────────────────────┤                                                                 
  │ 语义检索        │ 只用向量数据库                   │                                                              
  │                 │ 语义相似度排序                   │                                                              
  ├─────────────────┼───────────────────────────────┤                                                                 
  │ 全文检索        │ 只用关键词表                     │                                                              
  │                 │ 关键词匹配排序                   │                                                              
  ├─────────────────┼───────────────────────────────┤                                                                 
  │ 混合检索        │ 向量数据库 + 关键词表            │

  └─────────────────┴───────────────────────────────┘                                                                 
                                                                                                                      
  混合检索流程                                                                                                        
                                                                                                                      
  # 混合检索器配置                                                                                                    
  hybrid_retriever = EnsembleRetriever(                                                                               
      retrievers=[semantic_retriever, full_text_retriever],                                                           
      weights=[0.5, 0.5],  # 各占50%权重                                                                              
  )                                                                                                                   
                                                                                                                      
  用户提问

  │ 返回Top-K    │    │ 返回Top-K     │

           权重融合（0.5*A + 0.5*B）                                                                                  
                 ↓                                                                                                    
           最终排序返回
