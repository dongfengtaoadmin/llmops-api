# 工作流模块设计说明

## 工作流的本质

工作流本质上是一个 **LangChain 工具**，继承自 `langchain_core.tools.BaseTool`。相比普通工具，工作流具有更高的定制化能力，通过**可视化编排**的形式将多个功能节点组合在一起，实现复杂的业务逻辑。

核心实现位于 `internal/core/workflow/workflow.py`，使用 LangGraph 的 `StateGraph` 构建有向无环图（DAG）来完成节点编排。

## 节点类型

工作流支持以下节点类型（定义于 `internal/core/workflow/entities/node_entity.py`）：

| 节点类型 | 说明 |
|---------|------|
| `START` | 开始节点，整个工作流的入口，**有且仅有一个** |
| `END` | 结束节点，整个工作流的出口，**有且仅有一个** |
| `LLM` | 大语言模型节点，调用 LLM 完成对话/推理 |
| `TOOL` | 工具节点，调用已配置的工具 |
| `CODE` | 代码节点，执行自定义 Python 代码 |
| `DATASET_RETRIEVAL` | 数据集检索节点，从知识库检索相关内容 |
| `HTTP_REQUEST` | HTTP 请求节点，调用外部 API |
| `TEMPLATE_TRANSFORM` | 模板转换节点，进行变量替换和文本转换 |

## 什么是编排结束？

编排结束意味着工作流配置满足以下条件：

1. **节点完整性**
   - 存在唯一的开始节点（`START`）作为图结构的入口点
   - 存在唯一的结束节点（`END`）作为图结构的出口点

2. **图的连通性**
   - 所有节点都必须可达，不存在孤立节点
   - 使用 BFS 广度优先搜索验证连通性

3. **无环结构**
   - 使用 Kahn 拓扑排序算法检测，确保图中不存在环路
   - 工作流必须是有向无环图（DAG）

4. **边连接完整**
   - 节点之间通过边（Edge）正确连接
   - 边的 `source` 和 `target` 必须指向存在的节点
   - 支持并行边合并（多个节点指向同一个节点）

5. **数据引用正确**
   - 每个节点的输入数据只能引用其前置节点的输出
   - 使用逆邻接表获取前置节点，验证引用链路完整

## 工作流运行机制

```
用户输入 → 开始节点 → 节点1 → 节点2 → ... → 结束节点 → 输出结果
```

工作流通过 `_run()` 方法触发执行，使用 `StateGraph` 维护状态流转，支持流式输出每个节点的运行结果。
